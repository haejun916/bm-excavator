# 한국 B2C Pain Point 발굴 루틴

이 저장소를 클론한 Claude Code 루틴 세션이 매 실행마다 이 문서를 읽고 아래 작업을 수행한다.

## 목표
한국인이 일상에서 겪는 pain point(불편·불안·귀찮음·비용부담)를 최신 뉴스·실태조사에서
수집·분석해 B2C 비즈니스 기회를 발굴하고, 마크다운 리포트로 저장한다.

## 작업 순서

### 1. 데이터 수집 (RSS 전체 훑기 → 깊이 파기, 2단 구조)

수집은 **한국 언론사 통합 RSS를 폭넓게 훑어 후보를 추리고, 강한 것만 깊이 파는** 2단으로 한다.
검색 키워드에 갇혀 이미 아는 고통만 긁지 말고, 그날 새로 올라온 기사 전체에서 고통을 발굴한다.

**(1단계) feed_specs.csv의 모든 RSS를 읽어 전체 순회 수집**

이 저장소 안의 `data/feed_specs.csv`에 한국 언론사 RSS 원본 주소 약 155개가 정리돼 있다.
(knews-rss에서 가져와 우리 저장소에 직접 커밋해 둔 주소록. 남의 서비스에 의존하지 않는다.)
예: 경향신문 `https://www.khan.co.kr/rss/rssdata/total_news.xml` 같은 원본 피드 URL들.

**수집 절차:**
1. `data/feed_specs.csv`를 읽어 **모든 행의 RSS URL을 추출**한다(URL이 담긴 열 전체).
2. 추출한 RSS URL을 차례로 fetch해, 그날(또는 직전 실행 이후) 새로 올라온 기사의
   **제목·요약·링크·발행일**을 모은다.
   - 일부 피드는 응답이 없거나 형식이 깨질 수 있다. **실패한 피드는 건너뛰고** 계속 진행한다
     (한 피드 오류로 전체가 멈추지 않게).
   - 같은 기사가 여러 피드에 중복될 수 있으니 제목·링크 기준으로 **중복 제거**한다.
3. 노이즈 제거: 연예·스포츠·정치선동·단순 사건사고·국제토픽 등 **실생활 B2C 고통과 무관한
   기사는 버린다.** "사람들이 불편·불안·귀찮음·비용부담을 느끼는 생활 이슈"만 남긴다.
   (효율을 위해 csv에 `categories` 열이 있으면 society·health·economy·medical·women·culture
    분야를 **먼저** 보되, 특정 분야만 보는 게 아니라 전체를 훑는 것이 기본이다.)
4. 남은 기사에서 pain point **후보를 15~30개** 1차 추출한다(제목·요약 기반이라 가볍게).

> ⚙️ 분량 주의: 155개 피드를 매번 전부 받으면 시간·토큰이 큰다. 첫 실행이 너무 무거우면
> ① `categories`로 생활 관련 분야만 받도록 좁히거나 ② 매 실행 피드 일부만 로테이션하는 식으로
> 조정한다. 기본 목표는 "전체 순회"지만, 안정성이 우선이다.

**(2단계) 강한 후보만 원문으로 깊이 파기**
- 1차 후보 중 **고통이 세고 사업성이 보이는 8~12개**를 골라, 해당 기사 원문을 web_fetch로
  읽거나 web_search로 보강 검색해 **실태·규모·현재해결책·최근 제도변화**를 확인한다.
  (RSS는 보통 제목·요약만 주므로, 깊이는 이 단계에서 확보한다.)
- 필요하면 그 고통의 갈래를 더 파고드는 후속 검색을 1~2회 한다.

**(3단계) 시드 안전망 — 누락 영역 점검 / RSS 실패 시 폴백**
RSS가 특정 영역을 비우거나 접근이 막힌 경우를 대비해, 아래 7개 카테고리가 그날 리포트에서
하나도 안 다뤄졌으면 **그 카테고리만 골라 web_search로 1회씩** 보충한다(현재 연도를 붙여 검색).
돈·소비 / 건강·의료 / 육아·돌봄 / 노후·고령 / 1인가구·주거 / 직장·정신건강 / 기타 생활행정

### 2. Pain Point 추출 및 점수화
추출한 각 pain point마다:
- 카테고리: 돈 / 건강 / 육아 / 주거 / 직장 / 1인가구 / 기타
- 강도: 1~10
- 현재해결책: 없음 / 부분 / 있음
- **기회점수 = 강도 × (해결책이 '없음' 또는 '부분'이면 1.5, '있음'이면 1.0)**

**기회 선정 원칙(중요):**
- '있으면 좋은 것'이 아니라 '없으면 안 되는 것(돈 내는 고통)'에 가중치를 둔다.
- B2C 기회는 *단순 노동 대행*보다, **정보·전문성·접근권·신뢰**로 차별화되는 쪽을 우대한다.
  (단순 심부름/배달/줄서기는 단가 천장이 낮고 모방이 쉬워 감점)
- 각 기회에 "왜 지금인가(최근 규제·트렌드·제도 변화)"를 반드시 붙인다.

### 3. 리포트 저장
파일명: `reports/pain-point-{YYYY-MM-DD}-{아침|저녁}.md`
(한국시간 기준. 정오 이전 실행이면 '아침', 이후면 '저녁'. `reports/` 폴더가 없으면 만든다.)

**브랜치 규칙(반드시 지킬 것):**
- 저장·커밋·푸시는 **오직 `reports` 브랜치에서만** 한다.
- 작업 시작 시 `git checkout reports` 로 전환한다(원격에 없으면 `git checkout -b reports`).
- 리포트 파일을 추가한 뒤 `reports` 브랜치에 **직접 커밋·푸시**한다.
- **`main` 이나 새 `claude/...` 브랜치는 절대 만들거나 건드리지 말 것.**
- 커밋 메시지: "pain-point report {날짜} {시간대}"

## 리포트 형식 (이대로 출력)

```
# Pain Point 리포트 — {날짜} {시간대}

## 🔥 TOP 5 Pain Points
| 순위 | 내용 | 카테고리 | 강도 | 현재해결책 | 기회점수 |
|------|------|----------|------|-----------|----------|
(점수 높은 순 5행)

## 💡 B2C 기회 TOP 3
**1. [기회명] (점수: X)**
- Pain: ...
- 아이디어: ... (왜 단순노동이 아니라 차별점이 있는지 명시)
- 왜 지금인가: ...

(2, 3도 동일 형식)

## 📈 오늘 처음 등장한 트렌드
(불릿 2~4개. 직전 리포트와 비교해 새로 등장한 것 위주)

## 📊 수집 요약
(RSS 훑은 기사 수 / 1차 후보 수 / 원문 깊이 확인한 수 / 최종 추출 pain point 수)
```

## 누적 분석 (2회차 이후)
`reports/` 폴더에 이전 리포트가 있으면, 그중 최근 5개를 읽어
**여러 날 반복 등장한 pain point**를 리포트 맨 아래 `## 🔁 반복 등장 Pain Point` 섹션에
3줄 이내로 덧붙인다. (지속적으로 뜨는 고통 = 진짜 기회 신호)

## 카피라이트·품질 규칙
- 기사 문장을 그대로 복사하지 말 것. 모든 사실은 자기 문장으로 요약.
- 수치·통계는 출처가 분명한 것만 사용하고, 불확실하면 적지 말 것.
