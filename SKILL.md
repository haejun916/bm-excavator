# 한국 B2C Pain Point 발굴 루틴

이 저장소를 클론한 Claude Code 루틴 세션이 매 실행마다 이 문서를 읽고 아래 작업을 수행한다.

## 목표
한국인이 일상에서 겪는 pain point(불편·불안·귀찮음·비용부담)를 최신 뉴스·실태조사에서
수집·분석해 B2C 비즈니스 기회를 발굴하고, 마크다운 리포트로 저장한다.

## 작업 순서

### 1. 웹 검색으로 최신 데이터 수집 (동적 탐색 우선)

고정 키워드를 그대로 반복하지 말 것. **그날의 한국 생활 맥락에 맞는 검색어를 직접 만들어가며**
광범위하게 탐색하는 것이 목표다. 이미 알려진 고통만 긁지 말고 "새로 등장한 고통"을 찾는다.

**(A) 자유 탐색 — 기본 (최소 8~10회 검색)**
- "오늘 한국에서 사람들이 새롭게 불편·불안·분노·비용부담을 토로하는 생활 이슈"를 스스로 찾아라.
- 매 검색어는 **서로 다른 각도**여야 한다(같은 표현 반복 금지). 예: 계절/시기 이슈, 최근 가격·요금
  인상, 새 제도·규제로 생긴 혼란, 신규 서비스의 부작용, 특정 인구집단의 새 고충 등.
- 한 번 검색해서 흥미로운 고통의 단서가 나오면, 그 갈래를 **더 깊이 파고드는 후속 검색**을 한다.
- 시의성을 위해 "오늘/이번주/최근/2026" 같은 시간 한정어를 적극 활용한다.

**(B) 소스 다양화 — 뉴스 너머의 '날것'**
뉴스만 보면 이미 기사화된 늦은 고통만 본다. 아래 같은 곳의 *생생한 불만*도 함께 뒤진다.
- 커뮤니티·카페·게시판의 불만글, "이거 왜 이래" 류 토로
- 앱스토어·서비스 후기의 1~2점 리뷰(무엇이 불편한지)
- "○○ 대신 해주세요 / 어떻게 하나요" 류의 반복되는 도움 요청·외주 수요
- 새로 뜬 제품·서비스의 부작용·역효과

**(C) 시드 안전망 — 빠진 영역 점검 (각 1회 이상)**
자유 탐색이 특정 영역을 통째로 누락하지 않도록, 아래 7개 카테고리는 **최소 1회씩은**
그날의 관점에서 점검한다(단, 위 자유 탐색에서 이미 충분히 다룬 카테고리는 생략 가능).
돈·소비 / 건강·의료 / 육아·돌봄 / 노후·고령 / 1인가구·주거 / 직장·정신건강 / 기타 생활행정

**노이즈 제거 규칙:** 클릭베이트(혐한·국뽕·연예·정치선동), TikTok/YouTube 제목 낚시,
실생활 고통과 무관한 결과는 버린다. "실제로 돈을 낼 만한 고통"만 남긴다.

### 2. Pain Point 추출 및 점수화
추출한 각 pain point마다:
- 카테고리: 돈 / 건강 / 육아 / 주거 / 직장 / 1인가구 / 기타
- 강도: 1~10
- 현재해결책: 없음 / 부분 / 있음
- **기회점수 = 강도 × (해결책이 '없음' 또는 '부분'이면 1.5, '있음'이면 1.0)**

**기회 선정 원칙(중요):**
- '있으면 좋은 것'이 아니라 '없으면 안 되는 것(돈 내는 고통)'에 가중치를 둔다.
- B2C 기회는 *단순 노동 대행*보다, **정보·전문성·접근권·신뢰**로 차별화되는 쪽을 우대한다.
  (단순 심부름/배달/줄서기는 단가 천장이 낮고 모방이 쉬워 감점)
- 각 기회에 "왜 지금인가(최근 규제·트렌드·제도 변화)"를 반드시 붙인다.

### 3. 리포트 저장
파일명: `reports/pain-point-{YYYY-MM-DD}-{아침|저녁}.md`
(한국시간 기준. 정오 이전 실행이면 '아침', 이후면 '저녁'. `reports/` 폴더가 없으면 만든다.)

**브랜치 규칙(반드시 지킬 것):**
- 저장·커밋·푸시는 **오직 `reports` 브랜치에서만** 한다.
- 작업 시작 시 `git checkout reports` 로 전환한다(원격에 없으면 `git checkout -b reports`).
- 리포트 파일을 추가한 뒤 `reports` 브랜치에 **직접 커밋·푸시**한다.
- **`main` 이나 새 `claude/...` 브랜치는 절대 만들거나 건드리지 말 것.**
- 커밋 메시지: "pain-point report {날짜} {시간대}"

## 리포트 형식 (이대로 출력)

```
# Pain Point 리포트 — {날짜} {시간대}

## 🔥 TOP 5 Pain Points
| 순위 | 내용 | 카테고리 | 강도 | 현재해결책 | 기회점수 |
|------|------|----------|------|-----------|----------|
(점수 높은 순 5행)

## 💡 B2C 기회 TOP 3
**1. [기회명] (점수: X)**
- Pain: ...
- 아이디어: ... (왜 단순노동이 아니라 차별점이 있는지 명시)
- 왜 지금인가: ...

(2, 3도 동일 형식)

## 📈 오늘 처음 등장한 트렌드
(불릿 2~4개. 직전 리포트와 비교해 새로 등장한 것 위주)

## 📊 수집 요약
(검색 횟수 / 수집 기사·게시물 수 / 추출 pain point 수 / 자유탐색에서 새로 발견한 영역)
```

## 누적 분석 (2회차 이후)
`reports/` 폴더에 이전 리포트가 있으면, 그중 최근 5개를 읽어
**여러 날 반복 등장한 pain point**를 리포트 맨 아래 `## 🔁 반복 등장 Pain Point` 섹션에
3줄 이내로 덧붙인다. (지속적으로 뜨는 고통 = 진짜 기회 신호)

## 카피라이트·품질 규칙
- 기사 문장을 그대로 복사하지 말 것. 모든 사실은 자기 문장으로 요약.
- 수치·통계는 출처가 분명한 것만 사용하고, 불확실하면 적지 말 것.
